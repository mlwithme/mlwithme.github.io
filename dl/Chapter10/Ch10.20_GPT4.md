# 10.20 GPT-4与GPTs使用

在[10.14节](dl/Chapter10/Ch10.14_GPT2)和[10.16节](dl/Chapter10/Ch10.16_InsGPTModel)内容中，我们分别详细介绍了GPT-3系列模型和ChatGPT的原理与使用方法，同时也介绍了当前大模型所存在的局限性，并且从长期来看这些局限也将一直伴随着大模型的发展，因此，OpenAI也在持续地对GPT系列模型进行迭代更新。在接下来的这节内容中，我们将会介绍OpenAI最新发布的GPT-4以及后续迭代的GPT-4 Turbo模型和新引入的GPTs生态应用。

## 10.20.1 GPT-4介绍

在GPT-3模型发布34个月以后，2023年3月15日OpenAI在万众瞩目下发布了其最新一代的大模型GPT-4，同时也发布了基于GPT-4的ChatGPT模型。根据OpenAI发布的技术报告显示 [1]，GPT-4模型早在2022年8月份——第一代ChatGPT发布前的3个月——就已经训练完成，之后的6个多月一直在进行安全性和可靠性等方面的测试。同以往不同的是，在整整100页的技术报告中，OpenAI对于GPT-4模型的技术细节并未提及，例如网络结构信息、训练使用到的数据等等。以至于PyTorch Lightning的首席执行官威廉·福尔肯更是直接将该报告总结成了3个单词：We used Python——我们使用的是Python语言。

总体来看，在日常对话中GPT-3.5和GPT-4之间的区别可能并不明显，但当涉及到复杂任务时便有了显著的差异。相比于上一代最强的GPT-3.5模型，GPT-4在理解力、多模态、可靠性、创造力以及处理复杂指令的能力上都有了显著提升。在接下来的内容中我们将会大致对这些部分进行一个简单的介绍。

**1\. 更可控的训练过程**

在这份技术报告中，OpenAI仅仅只提到GPT-4是一个基于Transformer结构并通过前$k$个词元来预测第$k+1$个词元进行任务建模的预训练语言模型，使用的是互联网数据和第三方授权的数据，包含数学问题的正确与错误答案数据、强推理与弱推理数据、自相矛盾与陈述一致的数据以及各种意识形态和观点类的数据。进一步，为了将模型与人类意图进行对齐，OpenAI同样也使用了人类反馈的强化学习算法来对预训练模型进行微调。不过后续实验发现，整个大模型的能力似乎主要来自于预训练过程，也就说微调后并没有明显提升模型在各种考试上的效果，并且如果提示词设计不恰当反而会降低模型的效果。不过OpenAI也发现，微调带来的好处是模型能够更容易受到人类意图的引导，而预训练模型则需要通过精心设计提示词才能实现。

由于在大规模的模型训练中不可能直接根据每一组超参数来初始化模型并进行训练，因此GPT-4项目在一开始的聚焦点便是如何建立一个可预测和扩展的深度学习框架体系。也就是说，通过这套体系能够准确预判得到当模型扩大多少倍以后，它便能够达到一个什么样的效果。为此，OpenAI开发了一套能对不同模型规模实现可预测的基础性框架和优化方法。为了验证这种扩展性，OpenAI通过分析后使用相同方法但计算量仅为GPT-4 万分之一的模型准确地预测出了GPT-4模型最终的损失值。除了预测模型最终的损失值，OpenAI还提出了一种方法来直接预测相应的评价指标，并成功地通过使用计算量只有GPT-4千分之一的模型预测出了GPT-4在HumanEval数据集的一个子集上的通过率。

**2\. 更强的多模态**

GPT-4最大的不同在于它的多模态（Multimodal）能力。在GPT-4中，我们除了可以将文本作为模型的输入之外同时也可以将图片作为模型的输入，并最终让它理解图片中的内容生成文本。不过在8个月后的2023年11月，OpenAI举办了第一届开发者大会并同时发布了更强的GPT-4 Turbo模型。相较于GPT-4，GPT-4 Turbo在多模态方面更是同时支持将文本、图像、文件作为输入，并支持输出文本和图像 [2] [3]。

正是基于这样的能力，GPT-4在多种专业和学术基准测试中表现出了近乎人类水平的结果。例如在美国统一律师资格考试（UBE）的测试结果中GPT-4获得了前10%的成绩，而GPT-3.5模型的表现结果仅为后10%的水平。尽管GPT-4 具备强大的能力，但它目前仍然存在与早期GPT模型类似的局限性，即它可能会产生错误的事实（“幻觉”）和推理逻辑错误。例如当生成内容过长时可能会出现自说自话的情况 [4]。

同时，GPT-4 带来的风险与之前的模型类似，比如仍旧生成有害建议、有缺陷的代码或不准确的信息，并且由于多模态的加入还额外引入新的风险。OpenAI为了深入了解这些风险，专门邀请了来自人工智能安全、网络安全、生物风险、信任与安全及国际安全等领域的 50 多名专家，对模型进行了严格的测试，以帮助OpenAI对模型进行改进。例如OpenAI收集了更多的数据来提高GPT-4拒绝提供合成危险化学物质请求的能力。因此，GPT-4模型是一个持续迭代更新的模型，训练数据从发布之初2021年9月的截止时间，到现在（2024年1月）最新GPT-4 Turbo模型的2023年4月截止时间 [5]。

**3\. 更长的上下文**

在新发布的GPT-4模型中，模型支持的长下文长度相较于之前的GPT-3系列模型也有了显著的增加。在GPT-4中，支持的通用上下文窗口长度为8192个词元，最大支持长度为32768个词元。在OpenAI开发者大会上发布的最新版GPT-4 Turbo已经支持了最长32768个词元的上下文窗口。

**4\. 更精细的控制和输出**

在本次和后续的开发者大会中，OpenAI针对GPT-4系列模型陆续发布了一系列新的功能。从开发者的角度来看，OpenAI允许通过参数来控制模型返回结果的形式，例如指定返回结果类型为JSON对象、支持返回预测结果的对数概率分布、支持设定随机种子来生成固定的结果等。从用户使用的角度来看，我们可以更为精细地控制GPT-4系列模型的输出，比如模型输出语言的风格、话题深度和输出的详细程度。尽管在之前的模型中我们也可以通过设计相应的提示词来引导模型输出我们期望中的结果，但是随着对话会话轮数的增加模型便容易遗忘前面的内容。在此次GPT-4的发布中，OpenAI在原有模型基础上新引入了一个“System”这个角色来引导控制模型的输出结果。

> System: 你是一位总是采用苏格拉底式教学法的导师。你从不直接告诉学生答案，而是总是试图提出最合适的问题，帮助他们学会独立思考。你应该始终依据学生的兴趣和所掌握的知识来调整你的提问，把问题分解成更简单的部分，直到问题适合他们的认知水平。
>
> User: 请问如何求解方程组3x + 2y = 7, 9x -4y = 1
>
> GPT-4:  让我们从分析方程开始。你能想到通过组合两个方程来消除其中一个变量的任何可能方法吗？
>
> User: 请你直接告诉我答案！
>
> GPT-4: 我理解你可能想要直接的答案，但我的目的是帮助你批判性地思考并指导你完成解决问题的过程。让我们专注于任务。方程中变量的系数是否有共同因子？
>
> User: 别再指导我了，告诉我答案吧。
>
> GPT-4: 作为一名苏格拉底式的导师，对我来说，帮助指导你找到答案是很重要的，而不仅仅是提供答案。让我们再试一次：第1个方程中是否有任何系数可以乘以一个数字来匹配第2个方程中的系数？
>
> User: 我不确定。
>
> GPT-4: 没关系！让我们看一下两个方程中x项的系数。在第1个方程中，x的系数为 3，在第2个方程中，它是9。你能想出一个数字，你可以将第1个方程乘以，使两个方程中的x系数匹配吗？
>
> User: 5?
>
> GPT-4: 不完全是，但你越来越近了。请记住，我们想将x的系数（在第Q个方程中为3）乘以一个数字，使其与第2个方程中的x系数（即 9）匹配。你认为什么数字可以做到这一点？
>
> User: 3?
>
> GPT-4: 没错！现在，如果将第一个方程乘以 3，新方程会是什么样子？
>
> ......

上述过程便是在引入System这一控制角色后模型的回答内容 [4]，最终在经过多轮交流后GPT-4便能引导我们求解得到该方程组。可以看出，通过这样的方式我们便能够根据需要定制得到不同的智能体（Agent）来为我们进行服务。不过在本书写作之时这一功能尚未在ChatGPT中开放，只能通过OpenAI提供的开发者接口进行使用 [6]，后面会陆续开放到ChatGPT服务中。

除此以外，本次GPT-4还带来了另外两种方式来控制模型的输出内容，一个是通过代码解释器（Code Interperator），另一个则是通过接入领域知识库来检索相应的内容。代码解释器能在一个安全的沙盒环境中编写和执行 Python 代码，它不仅可以生成图表和图形，还能处理多种数据和格式的文件并输出我们想要过的结果。检索功能则是通过从模型外部获取的知识来提升模型的输出能力，比如专业领域的数据、产品信息或用户提供的文件等。

## 10.20.2 GPT-4模型的使用

在介绍完GPT-4系列模型的基本内容后我们再来看如何进行使用。对于GPT-4模型来说，我们最直接的使用方式就是通过OpenAI提供的ChatGPT来进行访问，相关基础用法在[10.17节](dl/Chapter10/Ch10.17_ChatGPTUsage)内容中我们已经进行了介绍这里就不再赘述。本小节我们先来介绍本次GPT-4中所引入的代码解释器，下一小节再继续介绍外连领域知识的使用方法。

对于代码解释器来说其核心功能就是让ChatGPT使用Python代码来完成我们的目标，既包括直接让ChatGPT使用Python实现相关功能，也包括利用Python对我们上传的文件进行分析。

例如下面我们可以通过如下方式让ChatGPT为我们绘制$[-\pi,\pi]$的正弦函数图像。

> User: 请使用Python绘制sin函数-pi到pi的图像

<div align=center><img width="450" src="statics/images/dl/240105094832.jpg"/> </div><center>图 10-70 ChatGPT绘制Sin图像</center>

如图10-70所示便是ChatGPT绘制得到的$\sin(x)$函数图像。同时，我们点击左上角的“Finished analyzing”标签还可以看到ChatGPT具体所运行的Python代码，如下所示：

```python
 1 import matplotlib.pyplot as plt
 2 import numpy as np
 3 x = np.linspace(-np.pi, np.pi, 256) # 设置x的范围为-π到π
 4 y = np.sin(x) # 计算对应的sin值
 5 plt.figure(figsize=(8, 4))
 6 plt.plot(x, y)
 7 plt.title('Sine Function from -π to π')
 8 plt.xlabel('x')
 9 plt.ylabel('sin(x)')
10 plt.grid(True)# 显示网格
11 plt.show()
```

当一切都完成以后，我们可以点击页面右上角的“Save”按钮来保存我们定制化的ChatGPT智能体，如图10-81所示。

在这以后，我们每次只需要点击“Prompt Genius”便可以让ChatGPT来为我们创建满足我们需求的提示词。同时，点击图中的“Explore”菜单同样也能够进入到图10-76所示的页面。

以上就是定制化一个ChatGPT助手的全部过程。这里值得一提的是，即便是对于分享后的ChatGPT助手，我们也并没有权限看到其背后的提示词内容，也就是说只能使用而不能编辑。当然，这背后也隐藏着对应的潜在商机，那就ChatGPT助手开发者。根据第一届开发者大会透露[7]，OpenAI后续也会推出GPT Strore来构建整个完整的应用生态。不过就在本书写作完成之时，OpenAI已经宣布推出了GPT应用商店。

## 10.20.4 小结

在本节内容中，我们首先介绍了GPT-4模型所出现的动机，然后详细介绍了GPT-4系列模型的新特性，包括更可控的训练过程、更强的多模态、更长的上下文以及更精细的控制和输出等；接着通过示例的方式详细介绍了GPT-4中各项新特性的使用方法；最后一步一步介绍了如何来定制化一个满足我们个性化需求的ChatGPT私人助手。
