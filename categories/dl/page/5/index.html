    梯度裁剪

    July 7, 2026
    Dl

    梯度裁剪#

    梯度裁剪（Gradient Clipping）是训练不稳定时的常用技巧：当梯度范数超过阈值时按比例缩放（或直接截断），防止梯度爆炸让参数更新步长过大。RNN / Transformer 等序列模型训练几乎必备。

    出现动机#

    训练 RNN 时梯度沿时间步反复累乘，一旦出现大幅梯度会让参数更新「飞出去」再也回不来，导致 loss 突然变成 NaN。梯度裁剪给梯度一个「天花板」：把范数过大或数值过大的梯度按阈值缩放，让参数更新始终在合理范围内——这是工程上「最便宜的稳定训练」技巧。

    优点缺点#

    优点：①实现简单（一行 torch.nn.utils.clip_grad_norm_），几乎所有 RNN / Transformer 项目默认启用；②有效缓解梯度爆炸与 loss 突刺；③对训练稳定性有明显收益，代价极小。缺点：①不能解决梯度消失；②阈值是超参，需要经验调节；③对 Adam 等自适应优化器作用相对有限，因为 Adam 本身已对梯度做归一化。

    术语别名#

    • Grad Clip

    BLEU

    July 7, 2026
    Ml, Dl

    BLEU#

    BLEU（Bilingual Evaluation Understudy）是 2002 年提出的机器翻译评估指标：通过统计「候选译文与参考译文之间 n-gram 的重合度」并辅以简短惩罚（Brevity Penalty）来打分。它是机器翻译、文本生成、摘要等任务最经典的自动评估指标之一。

    出现动机#

    早期机器翻译靠人工评估，昂贵且主观。BLEU 抓住「好的译文应该与人工参考译文共享大量 n-gram 短语」这一直觉，用统计 n-gram 重合度做自动评分；并对「短译文投机取巧」加惩罚。推出后成为 MT 领域的标准评估指标，至今仍被广泛使用。

    优点缺点#

    优点：①快速、自动、与人工评分相关性较好；②多年验证，几乎所有 MT / 摘要论文都报 BLEU；③实现简单，工程友好。缺点：①不评估语义，只看表面 n-gram 重合度；②对「同义改写」敏感度低；③已被 BLEURT、BERTScore、COMET 等基于 Embedding 的指标部分替代。

    术语别名#

    • BLEU
    • Bilingual Evaluation Understudy
    • BLEU 分数

    交叉熵

    July 7, 2026
    Ml, Dl

    交叉熵#

    交叉熵（Cross Entropy）是信息论中衡量「两个概率分布之间差异」的指标，也是分类任务最常用的损失函数。它表示「用分布 q 来编码真实分布 p 的样本所需的平均比特数」，最小化交叉熵等价于让模型预测分布逼近真实分布。

    出现动机#

    二分类用「对数损失」，多分类用「多类对数损失」——这些其实都是交叉熵的不同形式。它之所以成为分类任务的事实标准损失函数，是因为：①与 Softmax 配合，梯度形式极简；②最大化似然 = 最小化交叉熵；③对错误预测的惩罚力度合适，不会像 0-1 损失那样不连续。

    优点缺点#

    优点：①与 Softmax 输出层配合优雅，梯度直接 = 预测值 - 真实值；②最大化似然 = 最小化交叉熵，理论完备；③对错误预测惩罚力度大、收敛稳定。缺点：①在极度不平衡数据上会被多数类主导，需要配合 class_weight；②对「标签噪声」比较敏感（错误标签会显著拉大损失）；③与 focal loss 等加权变体相比，对难样本关注不足。

    • 多类交叉熵

    KL 散度

    July 7, 2026
    Ml, Dl

    KL 散度#

    KL 散度（Kullback-Leibler Divergence）是衡量两个概率分布 P 与 Q 之间差异的非对称指标：DKL(P‖Q) = Σ P(x) log(P(x)/Q(x))。它不是真正的「距离」（不满足对称性、三角不等式），但描述「用 Q 近似 P 时会丢失多少信息」。

    出现动机#

    很多场景需要比较两个分布：模型预测 vs 真实标签、Teacher vs Student、对真实分布的近似程度等。KL 散度给出一种信息论意义上的差异度量——「如果用 Q 来编码 P 的样本，平均会比用 P 编码多花多少比特」。它贯穿变分推断、模型蒸馏、RLHF 等多种现代方法。

    优点缺点#

    优点：①信息论意义清晰，与交叉熵关系明确（CE = Entropy + KL）；②对变分推断、模型蒸馏、强化学习等任务都是核心工具。缺点：①不对称：DKL(P‖Q) ≠ DKL(Q‖P)；②不满足三角不等式，不是严格的「距离」；③当 Q 接近 0 而 P 接近 1 时，KL 散度可能无穷大，需要数值稳定技巧。

    术语别名#

    信息熵

    July 7, 2026
    Ml, Dl

    信息熵#

    信息熵（Entropy）是信息论中度量「随机变量不确定性」的指标：H(P) = -Σ P(x) log P(x)。熵越大，不确定性越高；熵为 0 时事件完全确定。它是 KL 散度、交叉熵等概念的基础，也用于决策树特征选择（信息增益）。

    出现动机#

    不同事件携带的信息量不同：「太阳从东方升起」几乎没有信息量，「北京地震了」信息量很大。Shannon 把信息量定义为 log(1/P)，并用其期望值衡量「一个随机变量平均携带多少信息」——这就是熵。它不仅是信息论基础，也是机器学习中决策树、模型蒸馏、强化学习等多个场景的底层数学工具。

    优点缺点#

    优点：①度量不确定性 / 信息量的基础指标；②与 KL 散度、交叉熵构成信息论三件套；③决策树、信息增益等经典算法的核心。缺点：①对概率分布 P 的连续性敏感，需要做离散化；②对样本量小的分布估计不稳定；③只关注分布的「形状」，不关心具体事件。

    术语别名#

    • Entropy
    • 信息熵
    • Shannon Entropy
    • 香农熵

    PPO

    July 7, 2026
    Dl, Llm

    PPO#

    PPO（Proximal Policy Optimization，近端策略优化）是 OpenAI 在 2017 年提出的强化学习算法：通过「截断重要性采样比」让策略更新幅度始终在「近端」范围内，从而稳定训练。它是当前 RLHF（ChatGPT / InstructGPT）背后事实标准的优化器。

    出现动机#

    传统策略梯度（Policy Gradient）每一步都可能让策略发生巨大变化，训练极不稳定。TRPO 用 KL 散度约束策略变化，效果好但实现复杂。PPO 用一种更简单的「截断代理目标」实现同样的稳定效果——把更新限制在旧策略的附近，又不需要二次近似，被广泛采用。

    优点缺点#

    优点：①实现相对简单（一段损失函数）；②训练稳定性显著优于普通策略梯度；③在 RLHF、机器人、游戏 AI 等场景中都是默认选择。缺点：①对样本效率仍不如一些 off-policy 方法；②超参数（裁剪范围 ε、GAE λ 等）需要调节；③在大模型 RLHF 中，需要同时维护策略、价值、参考、奖励四个模型，工程复杂度高。

    术语别名#

    • PPO
    • Proximal Policy Optimization
    • 近端策略优化

    DPO

    July 7, 2026
    Dl, Llm

    DPO#

    DPO（Direct Preference Optimization，直接偏好优化）是 2023 年提出的 RLHF 替代方案：跳过奖励模型和强化学习环节，直接用「人对两个回答的偏好」数据训练语言模型。它把 RLHF 的两阶段训练压缩成单阶段，效果相当甚至更好。

    出现动机#

    RLHF 流程需要「监督微调 SFT → 训奖励模型 RM → PPO 强化学习」三步，工程复杂、训练不稳定、对算力要求高。DPO 的洞察是：把「奖励最大化 + KL 约束」的目标等价地改写成「偏好数据上的二元分类损失」——直接用偏好对 (chosen, rejected) 训语言模型，一步到位。

    优点缺点#

    优点：①无需训练奖励模型，工程链路大幅简化；②训练稳定性比 PPO 高得多；③效果在多数任务上与 RLHF 持平甚至更好。缺点：①对偏好数据质量依赖大（一条噪声数据可能误伤）；②对「超出偏好对分布」的泛化能力有限；③在极复杂推理任务上仍可能弱于精心调优的 RLHF。

    术语别名#

    • DPO
    • Direct Preference Optimization
    • 直接偏好优化

    Encoder

    July 7, 2026
    Dl

    Encoder-Decoder#

    Encoder-Decoder（编码器-解码器）是序列到序列建模的经典结构：Encoder 把输入序列压成上下文表示，Decoder 基于该表示逐步生成输出序列。Transformer 的原始论文采用的就是这种结构，至今仍是机器翻译、文本摘要、语音识别等任务的主流。

    出现动机#

    很多任务的输入和输出都是变长序列，且长度不一致：中文翻译成英文、文档生成摘要、语音生成文字。Encoder-Decoder 把「理解输入」与「生成输出」解耦：Encoder 把任意长度输入压成定长上下文（经过 Self-Attention + Cross-Attention），Decoder 在生成每一 token 时同时看「已生成内容」与「Encoder 输出」。

    优点缺点#

    优点：①通用框架，覆盖翻译 / 摘要 / 语音 / 问答等多种任务；②Encoder 与 Decoder 可以独立扩展（如 T5、BART）；③推理时 Encoder 只需跑一次，Decoder 自回归生成。缺点：①相比 Decoder-only 模型（如 GPT），结构更复杂；②训练数据量需求大；③对小模型 / 单任务，纯 Decoder 已能覆盖大部分需求。

    TextCNN#

    TextCNN 是 Kim 2014 提出的文本分类模型：使用一维卷积核对词向量序列做滑动窗口卷积，提取 n-gram 级别的局部特征，再通过最大池化和全连接层完成分类。结构简单、训练高效，是深度学习文本分类的经典 baseline。
