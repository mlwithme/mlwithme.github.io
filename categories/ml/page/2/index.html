    PCA

    July 7, 2026
    Ml

    PCA#

    主成分分析（PCA，Principal Component Analysis）是最经典的线性降维方法：通过正交变换把高维数据投影到一组按方差大小排序的新坐标系（主成分）上，保留前 k 个方差最大的方向，从而在低维空间里近似保留原始数据的主要结构。

    出现动机#

    高维数据（图像、基因、用户行为）既增加计算成本，又带来「维度灾难」——很多算法在高维下失效。PCA 的核心思想是「找几个最能把样本分开的轴」：把数据投影到这些轴上，既压缩了维度，又最大程度保留了信息（方差）。它既是降维工具，也是可视化（降到 2D / 3D）和去噪（丢弃方差小的方向）的基础。

    优点缺点#

    优点：①纯线性代数方法，计算高效，结果确定；②无监督，不需要标签，可用于任何数值型数据；③降维后的主成分可解释方差贡献率，便于筛选维度。缺点：①只能捕捉线性关系，对非线性流形（如环形、月牙形）效果差，需用 KPCA / t-SNE / UMAP 替代；②主成分的物理意义不直观，原始特征被线性组合后难以解读；③对特征尺度敏感，必须先做标准化。

    术语别名#

    • 主成分分析
    • 主成分降维

    July 7, 2026
    Ml

    朴素贝叶斯#

    朴素贝叶斯（Naive Bayes）是一类基于贝叶斯定理与「特征条件独立」假设的生成式分类模型：给定类别，先估计每个特征的条件概率，再由贝叶斯公式反推样本属于每个类别的后验概率，取最大的作为预测。它在文本分类、垃圾邮件识别等场景里非常经典。

    出现动机#

    现实里我们经常要根据多个特征推断一个类别（邮件是不是垃圾邮件、文章属于哪个主题）。贝叶斯定理天然适合「已知类别看特征」的反向推断；而「特征之间条件独立」的朴素假设虽然粗糙，却让模型参数从指数级降到线性级——这是它能用、且对文本任务特别有效的原因。

    优点缺点#

    优点：①训练只需一轮统计计数，速度极快；②对高维稀疏数据（文本词袋）效果稳健，是垃圾邮件分类的经典基线；③对缺失数据和少量样本不敏感。缺点：①「条件独立」假设在现实中几乎不成立，特征共现会让概率估计偏极端；②概率值受先验影响大，需要拉普拉斯平滑等手段；③对特征之间的相关性不敏感，在强相关特征上效果不如 LR / SVM。

    术语别名#

    • 朴素贝叶斯
    • NB
    • Bayes 分类器

    集成学习#

    集成学习（Ensemble Learning）通过组合多个「基学习器」（base learner）的预测，得到比单个学习器更准确、更稳定的最终模型。主要分为两大流派：Bagging（自助采样 + 多数投票，代表作随机森林）通过「多样性」降低方差；Boosting（顺序训练 + 纠错，代表作 AdaBoost、GBDT）通过「纠错」降低偏差。

    出现动机#

    单个模型往往有高方差（对数据敏感）或高偏差（拟合能力不足），难两全。集成学习的核心思想是「三个臭皮匠顶个诸葛亮」——用多个模型的集体智慧取长补短。Bagging 通过让基学习器尽量不同来降方差；Boosting 让基学习器串行纠错来降偏差。两条路线在 Kaggle 等竞赛中几乎统治了表格数据建模。

    优点缺点#

    优点：①通常能把单模型的精度再提升一档，竞赛中表现稳定；②Bagging 类天然并行，工程友好；③Boosting 类在表格数据上经常逼近神经网络的效果。缺点：①模型体积大，推理成本与基学习器数量成正比；②可解释性下降，难以回答「为什么这么预测」；③超参数多（基学习器数、学习率、树的深度等），调参复杂。

    术语别名#

    • Ensemble Learning
    • 集成学习

    • 模型融合

    • 随机森林

    • LightGBM

    AdaBoost

    July 7, 2026
    Ml

    AdaBoost#

    AdaBoost（Adaptive Boosting）是 Boosting 家族的奠基算法：顺序训练多个弱分类器（如浅层决策树桩），每轮训练时提高被前一轮错分样本的权重，让后续弱分类器重点关注难样本；最终预测时按各分类器在训练集上的准确率加权投票。

    出现动机#

    Boosting 的核心假设是「弱学习器可以叠加成强学习器」。AdaBoost 把这一思想落到工程细节：用一个标量权重 α 描述每个弱分类器的可信度，用样本权重 w 描述每个样本的难易程度；每轮训练后整体更新这两个权重，让模型沿着「错误样本逐步被纠正」的方向收敛。

    优点缺点#

    优点：①理论上只要基分类器略好于随机猜测，Boosting 就能逼近最优；②对噪声敏感度低于早期 Boosting 变体；③推导优雅，是理解 Boosting 思想的最佳入口。缺点：①对异常值仍然敏感，因为错分样本权重会被无限放大；②训练是顺序的，难以并行；③在特征维度极高、稀疏的场景下不如 Bagging / 深度模型。

    术语别名#

    • AdaBoost
    • Adaptive Boosting
    • 自适应提升

    TF-IDF

    July 7, 2026
    Ml

    TF-IDF#

    TF-IDF（词频 - 逆文档频率）是一种给词袋模型中每个词加权的方法：词频（TF）衡量「这个词在这篇文档里有多重要」，逆文档频率（IDF）惩罚「在很多文档里都出现的常见词」，两者相乘得到综合权重。它让「这个文档里很特别」的词在向量中获得更高权重。

    出现动机#

    纯词袋模型里，「的」「是」「了」这类高频词在每篇文档里都出现，对分类没贡献甚至有害。TF-IDF 通过 IDF 抑制这种「常见但无区分度」的词，让「仅在这篇文档里高频出现」的词（比如「量子」「财报」）在表示中更突出。这是搜索引擎最早期的核心技术，至今仍是文本特征工程的标配。

    优点缺点#

    优点：①权重公式简单直观，效果稳定，几乎是文本任务的默认 baseline；②和 BM25 等算法思路一致，可解释性强；③计算高效，适合大规模语料。缺点：①仍是词袋的稀疏表示，没法表达语义；②对长文档、短文档的归一化需要额外设计；③在语义相关、跨语言任务上不如 Embedding。

    术语别名#

    • TF-IDF
    • Term Frequency - Inverse Document Frequency
    • 词频 - 逆文档频率

    正则化

    July 7, 2026
    Ml, Dl

    正则化#

    正则化（Regularization）是在经验风险（训练损失）之外加一个「对模型复杂度的惩罚项」的总目标函数。常见形式有 L1（Lasso）、L2（Ridge / 权重衰减）、Dropout、早停等，目的是在拟合训练数据的同时约束模型复杂度，提升泛化能力。

    出现动机#

    模型在训练集上损失越低并不代表在新数据上效果越好——一味追求低训练损失会让模型「死记硬背」训练样本（过拟合）。正则化把「不要让模型太复杂」这件事量化成一个可加项，让优化目标在「拟合得好」与「不要太复杂」之间取得平衡。

    优点缺点#

    优点：①几乎所有模型都能用，代价低、收益大；②L1 还能做特征选择，得到稀疏解；③Dropout、Early Stopping 等实现简单，是深度学习训练的标配。缺点：①正则化系数（λ）需要调参，太大欠拟合、太小过拟合；②不同正则项对不同模型 / 任务效果差异大，需要经验；③Dropout 在推理时需要关闭，并对应调整输出期望，工程细节容易踩坑。

    • L2 正则化（Ridge / 权重衰减）
    • Weight Decay

    特征缩放

    July 7, 2026
    Ml

    特征缩放#

    特征缩放（Feature Scaling）是把不同量纲、不同量级的特征统一到相近数值范围的预处理操作，常见方法有：标准化（Standardization，让均值为 0、方差为 1）、归一化（Normalization，把数值压到 [0,1]）。它是 KNN、SVM、神经网络等依赖距离或梯度模型的必要前置步骤。

    出现动机#

    现实特征往往量纲差异巨大：年龄 0–100、收入 0–100 万、点击率 0–1。如果不缩放，距离度量、梯度方向都会被「收入」这种大数主导，模型学偏。缩放把所有特征拉到同一数量级，让每个特征对模型的影响公平。

    优点缺点#

    优点：①显著提升 KNN、SVM、神经网络等模型的收敛速度与稳定性；②不同特征之间具备可比性，便于业务解读；③和 PCA、KMeans 等基于距离的方法配合效果更好。缺点：①对决策树、随机森林这类基于阈值的模型基本无影响，反而可能损失可解释性；②训练集 / 测试集要用同一套缩放参数（fit on train），否则会引入数据泄露；③对极端值（长尾分布）敏感，可考虑 RobustScaler。

    术语别名#

    半监督学习#

    半监督学习（Semi-Supervised Learning）是一类同时利用「少量有标签数据」与「大量无标签数据」进行建模的学习范式。常见方法包括 Self-Training（用模型预测无标签样本作为伪标签再训练）、Co-Training（两个视图互相打标）、Label Propagation（图上的标签传播）等。

    出现动机#

    标注数据成本高（医学影像需要专家标注、文本任务需要人工分类），但无标签数据极易获取。半监督学习的核心假设是「相似样本应共享标签」，借助数据分布的内在结构，把无标签样本的丰富信息传递给模型，从而在标注极少的情况下也能训练出效果不错的模型。

    优点缺点#

    优点：①显著降低对标注数据的依赖；②在数据量充足但标注不足的真实业务里，往往比纯监督学习更稳健；③可以与监督学习、主动学习结合使用。缺点：①对「聚类假设 / 流形假设」依赖较强，若数据不满足这些假设反而会引入噪声；②伪标签一旦出错会被反复放大，需要设置置信度阈值或剪枝；③理论分析困难，超参数选择依赖经验。

    Word2Vec#

    Word2Vec 是 Mikolov 等人在 2013 年提出的词向量训练模型：通过 CBOW（用上下文预测中心词）或 Skip-gram（用中心词预测上下文）两种浅层神经网络结构，把每个词映射成一个固定维度的稠密向量，让语义相近的词在向量空间里距离相近。它是预训练词向量时代的代表工作。

    出现动机#

    早期的词表示是 one-hot 编码（词表大小 = 向量维度，稀疏且无语义），没法直接喂给模型做相似度计算。Word2Vec 的关键洞察是「一个词的语义可以由它周围的上下文来定义」——通过让模型在大量文本上学「上下文 ↔ 中心词」的关系，得到的词向量天然具备「king - man + woman ≈ queen」这种语义运算能力。

    优点缺点#

    优点：①训练快、效果好，是早期 NLP 任务的标配；②词向量语义关系可线性运算，可解释；③工程实现简单，有 C / gensim / PyTorch 多种实现。缺点：①静态词向量，一词多义（如「bank」）无法区分；②对 OOV（词表外）词没有表示；③效果已被后续的 ELMo、BERT 等上下文相关词向量全面超越。

    GloVe#

    GloVe（Global Vectors for Word Representation）是斯坦福在 2014 年提出的词向量训练方法：基于全局词-词共现矩阵的统计信息，把每个词映射为低维向量，使「两个词向量的点积 ≈ 它们共现概率的对数」。它是 Word2Vec 之外另一条「静态词向量」的经典路线。

    出现动机#

    Word2Vec 只用局部滑动窗口训练，忽略了整个语料的全局统计信息。GloVe 抓住这一空白：先构建词-词共现矩阵 X（Xᵢⱼ 表示词 i 与词 j 在某窗口内共同出现的次数），再让词向量的点积拟合「共现概率比」的对数——既保留局部信息（窗口），又利用全局统计（矩阵）。

    优点缺点#

    优点：①利用全局共现统计，训练出的词向量在小语料上更稳定；②与 Word2Vec 性能相当，但训练目标更直观；③在词相似度、类比任务上效果稳健。缺点：①仍是静态词向量，不解决一词多义；②矩阵分解代价高，训练对内存不友好；③已被 BERT、GPT 等动态词向量取代。
