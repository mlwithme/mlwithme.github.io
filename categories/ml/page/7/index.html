    梯度下降

    July 7, 2026
    Ml, Dl

    梯度下降#

    Gradient Descent（梯度下降）是一类通过「沿目标函数负梯度方向迭代更新参数」寻找最小值的优化算法，是几乎所有机器学习与深度学习模型训练的核心机制。

    出现动机#

    很多模型的损失函数没有闭式解（神经网络），或闭式解代价太高。梯度下降提供通用做法：先随机初始化参数，反复沿「让损失下降最快」的方向迈一小步，期望逐步逼近局部最小值。

    优点缺点#

    优点：①通用、简单，几乎适用所有可微损失函数；②配合反向传播可扩展到超大规模神经网络；③有大量变体（Momentum、Adam、RMSProp）解决收敛与稳定性问题。缺点：①对学习率敏感，太大不收敛、太小收敛慢；②只能保证收敛到局部最优 / 鞍点；③存在梯度消失 / 爆炸等病态情况。

    术语别名#

    • Gradient Descent
    • 梯度下降
    • GD
    • SGD

    相关术语#

    • 学习率
    • Momentum
    • Adam
    • 反向传播

    过拟合

    July 7, 2026
    Ml

    过拟合#

    Overfitting（过拟合）是模型「死记硬背」训练数据、在新数据上表现很差的现象。常见于模型复杂度高、训练数据不足 / 噪声大、训练轮数过多等场景。

    出现动机#

    训练模型的目标是「在新数据上也能答对」，但一味追求训练集低损失会让模型去拟合噪声和偶然规律，反而损害泛化能力。过拟合是 ML 最经典、最常遇到的病态之一。

    优点缺点#

    优点（识别角度）：①训练 loss 持续下降、验证 loss 先降后升，是过拟合的典型信号；②通过正则化、早停、Dropout、数据增强可缓解；③理解过拟合就理解了偏差 / 方差权衡。缺点（问题角度）：①会显著降低模型在新数据上的表现；②对样本量小的任务特别严重；③有时与「欠拟合」难以一眼区分。

    术语别名#

    • Overfitting
    • 过拟合
    • Overfit

    相关术语#

    • 欠拟合
    • 正则化
    • 偏差与方差

    欠拟合

    July 7, 2026
    Ml

    欠拟合#

    Underfitting（欠拟合）指模型过于简单，无法捕捉数据中的潜在模式，导致训练集和测试集上表现都不好。是与过拟合相对的另一端问题。

    出现动机#

    模型能力不足时，既学不到训练数据的规律，也泛化不到测试数据。常见原因包括模型太简单、特征不足、训练时间不够、正则化过强等。

    优点缺点#

    优点（识别角度）：①训练 / 验证 loss 同时居高不下，是欠拟合的典型信号；②可通过换更复杂模型、增加特征、减少正则化等手段缓解。缺点（问题角度）：①模型完全无法捕获数据中的有效信号；②调参或换模型都见效慢；③在某些情况下，强行增加复杂度又会过拟合，需要在偏差 / 方差间权衡。

    术语别名#

    • Underfitting
    • 欠拟合
    • Underfit

    数据集

    July 7, 2026
    Ml

    数据集#

    Dataset（数据集）指用于训练、测试、验证 AI 模型的样本集合。它是 ML 的「燃料」——模型能力的天花板很大程度上由数据集规模、多样性、标注质量决定。

    出现动机#

    ML 模型学到的所有「知识」都来自数据集。数据集的规模、质量、分布直接决定模型的表现。设计良好的数据集 + 合适的模型 + 充分的训练，是 ML 项目成功的三大支柱。

    优点缺点#

    优点：①结构化数据让训练 / 评估可重复；②公开数据集（如 ImageNet、COCO、SQuAD）让社区共享与对比成为可能；③数据版本管理（DVC、lakeFS）让迭代可追溯。缺点：①高质量数据集标注成本高；②数据偏见会传递给模型；③数据隐私与合规要求越来越严。

    术语别名#

    • Dataset
    • 数据集
    • Data Set

    训练集

    July 7, 2026
    Ml

    训练集#

    Training Set（训练集）是数据集中用于「教」模型学习规律的部分，通常占整体数据的 60–80%。模型参数在训练集上反复迭代更新，最终学到「输入 → 输出」的映射。

    出现动机#

    ML 模型学习的目标是从训练数据中提取统计规律。训练集越大、越多样、标注越准，模型学到的「知识」就越扎实。训练集与验证集 / 测试集的「隔离」也是评估泛化能力的前提。

    优点缺点#

    优点：①直接决定模型学到的知识；②量大即可学到丰富的统计模式。缺点：①过大训练集会消耗大量算力；②训练集上的低 loss 不代表测试集上也好（过拟合）；③对样本分布敏感，分布外样本效果差。

    术语别名#

    • Training Set
    • 训练集
    • Train Set

    相关术语#

    • 测试集

    验证集

    July 7, 2026
    Ml

    验证集#

    Validation Set（验证集）是数据集中用于「调超参数、监控训练过程、提前停止训练」的部分。它不直接参与梯度更新，但用于选择最佳模型 / 超参组合。

    出现动机#

    模型选择本身是「元训练」：选学习率、选网络深度、选正则化强度。如果用测试集来做选择，相当于「让测试集参与了训练」，评估结果会偏乐观。验证集专门承担这个角色。

    优点缺点#

    优点：①把「模型选择」与「最终评估」分开，结果更可靠；②可用于早停（Early Stopping），避免过拟合；③可与超参搜索（Grid / Bayesian）配合。缺点：①数据被进一步切分，每一部分变小，统计噪声变大；②对时间序列数据需要按时间顺序切分；③可与训练集做交叉验证（CV）以更稳健地评估。

    术语别名#

    • Validation Set
    • 验证集
    • Dev Set
    • 开发集

    相关术语#

    • 训练集

    测试集

    July 7, 2026
    Ml

    测试集#

    Test Set（测试集）是数据集中「完全未被模型见过、用于最终评估」的部分。它是模型泛化能力的真实写照——只在训练结束、模型冻结后使用一次。

    出现动机#

    ML 项目的最终目标是「在新数据上也能答对」。测试集模拟这种「真实新数据」：模型在训练过程中完全没见过它，评估结果才能反映真实泛化能力。

    优点缺点#

    优点：①提供无偏的最终评估指标；②模拟真实部署场景；③用于跨模型 / 跨方法的公平对比。缺点：①一旦「偷看」测试集，评估就失去意义；②样本量有限，对小数据集噪声大；③与训练 / 验证集的分布需要一致才公平。

    特征

    July 7, 2026
    Ml

    特征#

    Features（特征）是数据中的「单个可测量属性 / 变量」，即模型的输入。在表格数据里是列名，在图像里是像素值，在文本里是词向量——决定模型「看得到什么」。

    出现动机#

    ML 模型的输入就是特征向量。特征质量决定模型上限——垃圾特征输入，再强的模型也只能学到垃圾规律。「特征工程」（构造、选择、转换特征）长期以来是 ML 工程师的核心工作。

    优点缺点#

    优点：①把原始数据转成模型可用的形式，是 ML 流水线的关键一步；②特征工程能显著提升模型效果，往往比换模型更划算；③现代深度学习能自动学特征，降低人工成本。缺点：①特征工程耗时大、依赖领域知识；②特征数量过多会导致「维度灾难」；③不同模型对特征的尺度 / 分布要求不同（线性模型需归一化，树模型不需要）。

    术语别名#

    • Features
    • 特征

    • 输入变量

    标签

    July 7, 2026
    Ml

    标签#

    Labels（标签）是监督学习中「正确的输出 / 类别」，即每个样本对应的「标准答案」。模型通过最小化预测与标签的差异来学习参数。

    出现动机#

    监督学习的训练目标是「预测尽可能接近标签」。标签是训练信号的来源：没有标签，监督学习就无从谈起。标签可以是类别（分类）、数值（回归）、序列（机器翻译）、结构（命名实体识别）等。

    优点缺点#

    优点：①为监督学习提供明确的学习目标；②标注质量越高，模型上限越高。缺点：①标注成本高：医学、法律等领域需要专家；②标签噪声会显著损害模型；③对类别分布不平衡敏感，需要特殊处理；④标签定义本身可能有歧义（不同标注员可能给出不同答案）。

    归一化

    July 7, 2026
    Ml

    归一化#

    Normalization（归一化）是把数值特征按比例缩放到标准范围（通常是 [0, 1]）的预处理方法。它防止「数值大的特征主导模型」，是 KNN、SVM、神经网络等依赖距离或梯度模型的必要前置步骤。

    出现动机#

    现实特征量纲差异巨大：年龄 0–100、收入 0–100 万、点击率 0–1。不归一化的话，距离度量、梯度方向都会被「收入」主导，模型学偏。

    优点缺点#

    优点：①显著提升 KNN、SVM、神经网络等模型的收敛与稳定性；②特征之间具备可比性；③和 PCA、KMeans 等距离方法配合效果好。缺点：①对树模型基本无影响；②训练 / 测试集要用同一套缩放参数；③对极端值敏感，可考虑 RobustScaler。
