    强化学习#

    Reinforcement Learning（强化学习，RL）是一类让 AI「智能体」通过与环境交互、试错、并根据奖励或惩罚信号调整行为的学习范式。它是 AlphaGo、机器人控制、游戏 AI 等场景的核心方法，也是 RLHF 等大模型对齐技术的基础。

    出现动机#

    监督学习需要大量标注样本，但很多场景下没有「正确答案」——只有「做得好不好」的反馈信号（如下棋的输赢、对话的用户满意度）。强化学习直接针对这种「序贯决策」问题：通过与环境交互，根据累计奖励信号学习最优策略。

    优点缺点#

    优点：①无需标注样本，适合环境可交互但难以标注的场景；②可以学到超出人类的策略（如 AlphaGo）；③是当前大模型对齐（RLHF）的核心组件。缺点：①训练样本效率低，往往需要海量交互；②奖励函数设计困难，错误奖励会导致「reward hacking」；③训练稳定性差，超参数敏感。

    机器学习#

    术语解释#

    机器学习（Machine Learning，ML）是 AI 的核心子领域：通过让系统从数据中学习规律与模式，而不是显式编程，来完成任务。它包含监督学习、无监督学习、半监督学习、强化学习等范式，是当代 AI 落地的最主要路径。

    显示编程于隐式编程

    关于到底什么是机器学习（Machine Learning），可能不同的人会有不同的理解，自然也就产生了不同的定义。下面我们主要介绍一下计算领域内两位大师对于什么是机器学习所给出的定义。

    第一位是人工智能先驱亚瑟·塞缪尔（Arthur Samuel），他在1959年创造了“机器学习”一词 [1]。塞缪尔认为，所谓机器学习是指： 计算机能够具备根据现有数据构建一套不需要进行显式编程的算法模型来对新数据进行预测的能力。这里所谓不需要进行显式编程便是区别于传统程序算法需要人为指定程序执行步骤的过程。

    Field of study that gives computers the ability to learn without being explicitly programmed

    第二位是卡内基梅隆大学的计算机科学家汤姆·迈克尔·米切尔（Tom Michael Mitchell），他给出了一个相较于塞缪尔更加正式与学术的定义。米切尔认为，如果计算机程序能够在任务T中学得经验E，并且可以通过指标P进行评价，同时根据经验E还能够提升程序在任务T中的评价指标P，这就是机器学习[2]。这段话对于初学者来讲稍微有点拗口，其实际想要表达的就是，如果一个计算机程序能够自己根据数据样本学习，以此获得经验并通过迭代逐步提高最终的表现结果，则这个过程就被称为机器学习。

    A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E.

    监督学习#

    Supervised Learning（监督学习）是用「已标注」的训练数据学习输入到输出映射的机器学习范式：每个样本都给出「正确答案」（标签），模型通过最小化预测与标签的差异来调整参数。

    出现动机#

    很多任务有明确的「正确答案」：邮件是不是垃圾、图像里是猫还是狗、房价是多少。监督学习把这种「有标注数据 → 学映射」问题形式化，是目前应用最广泛、效果最稳健的机器学习范式。

    优点缺点#

    优点：①目标清晰（最小化预测误差），训练稳定；②在数据充足时往往拿到当前最优效果；③理论成熟，调参经验充足。缺点：①依赖大量高质量标注样本，标注成本高；②模型只学到训练分布内的模式，分布外样本表现差；③难处理「没有标签」的场景（如聚类、异常检测）。

    无监督学习#

    Unsupervised Learning（无监督学习）是用「无标注」数据训练模型，让算法自行发现数据中的隐藏结构与模式的学习范式。常见任务包括聚类、降维、密度估计、自监督表征学习等。

    出现动机#

    真实世界里大量数据都没有标签——用户行为日志、网络流量、基因表达等。给这些数据全部人工打标成本极高。无监督学习用算法自动发现数据内在规律，是数据探索与预处理的利器，也是大模型自监督预训练的核心思想。

    优点缺点#

    优点：①无需标注数据，可用规模几乎无限；②能发现人类难以直观看到的隐藏模式；③为聚类、降维、异常检测等任务提供基础工具。缺点：①没有「正确答案」，评估困难；②学到的「结构」是否符合业务预期需要人工校验；③效果通常弱于有监督场景。

    分类#

    Classification（分类）是机器学习的核心任务之一：把样本分到预定义的若干类别中，如「邮件是垃圾 / 非垃圾」「图像是猫 / 狗 / 鸟」。常见算法包括逻辑回归、SVM、决策树、随机森林、神经网络等。

    出现动机#

    现实里有大量「分到固定类别」的需求：用户画像分群、信用卡欺诈识别、医学影像诊断。分类任务的目标清晰（输出离散标签），评价指标成熟（准确率、精确率、召回率、F1），是 ML 中应用最广泛的任务之一。

    优点缺点#

    优点：①任务定义清晰、数据标注相对容易；②评价指标丰富；二分类 / 多分类 / 多标签变体可应对不同需求；③算法选择多，从线性到深度模型都有成熟方案。缺点：①对类别不平衡敏感，多数类会主导模型；②对标签噪声敏感；③高度依赖特征工程或 Embedding 质量。

    回归#

    Regression（回归）是机器学习的核心任务之一：预测一个连续的数值输出，如房价、销量、温度、用户停留时长。常见算法包括线性回归、多项式回归、决策树回归、神经网络回归等。

    出现动机#

    现实里有大量「预测一个数」的需求：电商销量预测、股票价格预测、天气温度预测。回归任务输出连续值，损失函数（MSE、MAE）和评估指标（MSE、RMSE、R²）也相应连续化。

    优点缺点#

    优点：①任务定义清晰，输出数值可直接解释；②损失函数连续可微，便于优化；③线性回归 / 决策树回归等基线模型解释性强。缺点：①对异常值敏感，单个离群点可能显著影响模型；②对非线性关系的建模需要特征工程或非线性模型；③预测区间难以估计，常需要额外的不确定性建模。

    聚类#

    Clustering（聚类）是无监督学习的核心任务：把相似样本自动归到同一组，让组内相似、组间差异。常见算法有 K-Means、层次聚类、DBSCAN、谱聚类等。

    出现动机#

    在没有标签的情况下，人们也想从数据中发现「分组」：客户分群、商品归类、文档主题、基因表达聚类等。聚类算法让机器自动找到这种「内在结构」，是探索性数据分析与推荐系统的常用工具。

    优点缺点#

    优点：①无需标注数据，可直接对原始样本运行；②算法选择多，K-Means 速度快、DBSCAN 能找任意形状、层次聚类有层次树；③结果可视化直观，配合业务解读性强。缺点：①没有「正确答案」，评估困难；②算法对参数 / 数据分布敏感；③聚类数 K 多数场景下需要人工指定。

    支持向量机#

    Support Vector Machine（支持向量机，SVM）是一类以「最大间隔」为核心的几何分类模型：在特征空间里寻找一个超平面把两类分开，并最大化两类样本到超平面的最小距离（margin）。核函数让 SVM 能处理非线性问题。

    出现动机#

    线性分类器可以分开的样本很多，但「分得好不好」差别很大——分界线离样本越远，模型对未来数据鲁棒性越强。SVM 把这条直觉形式化为「最大间隔」准则，并推导出对偶问题，可以用核函数处理非线性映射。

    优点缺点#

    优点：①最大化间隔带来良好泛化能力，小样本上往往表现优异；②通过核函数可隐式处理高维映射；③决策只依赖少数「支持向量」，模型稀疏。缺点：①对大规模数据训练慢，需要 SMO 等专门优化；②对噪声 / 重叠类敏感，需要软间隔 + C 调参；③核函数与参数选择依赖经验。
