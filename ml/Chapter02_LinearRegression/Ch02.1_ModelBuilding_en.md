## 2.1.1 Understanding Linear Regression Models

Generally speaking, each algorithm in Machine Learning is born to solve a certain type of problem. In other words, there are indeed some problems in real situations that can be solved by linear regression, such as predicting housing prices. But some may ask, why should linear regression be used for predicting housing prices instead of other algorithmic models? The reason lies in the fact that common sense tells us that housing prices increase with the growth of area and generally show a linear growth trend. So, is there a situation where the price decreases after the area reaches a certain level, thus not conforming to linear growth?This may certainly exist, but in actual processing, we will definitely prioritize the linear regression model. When the results are not satisfactory, we will try other algorithms. Therefore, after learning multiple algorithms, when faced with a specific problem, we need to consider which model is more suitable for solving the current problem.

## 2.1.2 Build a linear regression model

Generally speaking, when we encounter a practical problem, we first select an appropriate model based on the background of the problem and common sense. Meanwhile, common sense now tells us that the growth of housing prices is more likely to follow models such as linear regression. Therefore, we can consider building a linear regression model as shown below.

$$
\hat{y}=h(x)=wx+b\tag{2-1}
$$
Among them, $w$ is called the weight parameter (Weight), and $b$ is called the bias (Bias) or intercept (Intercept). After the unknown parameters $w$ and $b$ are solved by a certain method, it means that we have obtained this prediction model, that is, given a house area $x$, we can predict its corresponding house price $\hat{y}$.

Note: In Machine Learning, the so-called model can be simply understood as a function.

## 2.1.3 Solving the Linear Regression Model

Once a model is established, the natural question that comes to mind is how to solve the model $h(x)$ using the given data, also known as the training set (Training Data). In middle school, we learned how to find the equation of a line passing through two coordinate points, but in the above scenario, this approach clearly won't work (since not all points lie on a single line). So, is there a good solution?

At this point, we need to shift our thinking. Since we cannot directly solve for it, let's try an indirect approach. Now, imagine what condition $h(x)$ must satisfy to be considered a good $h(x)$? Recall the purpose of solving for $h(x)$. Isn't it to output an "accurate" housing price $\hat{y}$ after inputting the area $x$? Since directly solving for $h(x)$ is difficult, let's start from "accuracy". 

But how do we define accuracy? Here, we can characterize "accuracy" by calculating the mean squared error between the true housing price and the predicted housing price for each sample, that is 


$$
\begin{cases}
 J(w,b)=\frac{1}{2m}\sum\limits_{i=1}^{m}{{{({{y}^{(i)}}-{{{\hat{y}}}^{(i)}})}^{2}}}\\[2ex] 
  {{{\hat{y}}}^{(i)}}=h({{x}^{(i)}})=w{{x}^{(i)}}+b  
\end{cases}
\tag{2-2}
$$
Here, $m$ represents the number of samples; $x^{(i)}$ represents the value of the $i$-th sample, which is the area of the $i$-th house; $y^{(i)}$ represents the true price of the $i$-th house; $\hat{y}^{(i)}$ represents the predicted price of the $i$-th house. 

From Equation (2-2), we know that the parameters $\hat{w}$ and $\hat{b}$ when the function $J(w,b)$ takes its minimum value are the target parameters we are looking for. Why? Because when $J(w,b)$ takes its minimum value, it means that the error (Error) between the predicted values and the true values of all samples is minimized at this time. If we take it to the extreme, when all predicted values are equal to the true values, then $J(w,b)$ at this time is 0.Therefore, the problem of how to solve the model $$h(x$$ is transformed into the problem of how to minimize the function $J(w,b)$, and $J(w,b)$ also has a specific term called the objective function, cost function, or loss function.

## 2.1.6 Example Code for Linear Regression

After installing the corresponding Python packages, you can formally model and solve linear regression. The detailed and complete code can be found in the ` Book/Chapter02/01_house_price_train.py ` file. 

**1. Import packages**

First, you need to import the relevant Python packages to be used. The code is as follows: 

```Python
1 import numpy as np
2 from sklearn.linear_model import LinearRegression
3 import matplotlib.pyplot as plt
```

**2. Create samples (dataset)**

Create a dataset for training the model, the code is as follows: 

```Python
1 def make_data():
2     np.random.seed(20)
3     x = np.random.rand(100) * 30 + 50  # Square
4     noise = np.random.rand(100) * 50
5     y = x * 8 - 127 - noise            # Pirce
6     return x,y
```

In the above code, the purpose of line 4 is to add a certain amount of noise (error) to the dataset to simulate real-world situations. 

**3. Define the model and solve it**

Solve and predict the parameters of the linear regression model using the ` LinearRegression ` class in sklearn. The code is as follows: 

```Python
1 def main(x, y):
2     model = LinearRegression()         
3     x = np.reshape(x, (-1, 1))
4     model.fit(x, y) # training
5     y_pre = model.predict(x) # prediction
6     print(y_pre)
```

In the above code, line 2 ` np.reshape(x,(-1,1)) ` means transforming ` x ` into ` [n,1] ` shape. As for ` n ` what exactly it is, it will be deduced by ` np.reshape ` function itself. For example, if the row shape of a is [4,5], and you want to change a to [2,10] shape, you can use ` a.reshape([2,10]) `, or use ` a.reshape([2,-1]) ` to perform shape transformation. Line 4 is to solve for the parameters in the model through ` fit() ` method, that is, $w$ and $b$ in equation (2-1). Line 5 predicts the price of a house based on its area.

**4. Running Results**

Finally, call the defined function to run the program and output the parameter results obtained from the final training. The sample code is as follows: 

```Python
1 if __name__ == '__main__':
2     x, y = make_data()
3     main(x, y)                        
4     #参数w=[7.97699647],b=-154.31885006061555
5     #面积50的房价为： [244.53097351]
```

From the above output results, it can be found that the parameters are $w = 7.97$ and $b = -154.32$, which means that $h(x) = 7.97x - 154.32$. After that, new inputs can be predicted using $h(x)$. Meanwhile, the corresponding fitted line can also be drawn based on the solved model, as shown in Figure 2-6.
